## What's actually in here

- `gpt-2/` — a customized fork of OpenAI's original TF1 GPT-2 repo (`model.py`, `encoder.py`,
  `sample.py`, `download_model.py` are stock OpenAI files). Two custom entrypoints:
  - `main.py` — interactive sampling, with a custom deterministic seed derived from wall-clock
    time (`fortminute()`: divides the UTC day into units of 864 seconds, ~0.01 of a day — a
    decimal-time seed, not a random one). Time-seeded generation ties naturally to
    astrological/transit framing (see below).
  - `kjv.py` — same structure, presumably pointed at a KJV-Bible-trained checkpoint (not
    included; `models/` isn't in this checkout).
  - `pemdas.py` / `hollow.py` — near-duplicate utility modules (converted from a Jupyter
    notebook, still carries `# In[17]`-style cell markers) with the actual reusable logic:
    - `squished(s)` — uppercase, strip non-alphanumerics, collapse consecutive duplicate
      characters.
    - `MTRXTWER(s)` ("matrix tower") — squish, then print one character per line.
    - `PRINTWR(s)` — squish, then print in fixed-width rows (default 3 chars/row, padded with
      `X`/`XZ`), forming a literal text "tower." **This is what "the magic tower transformer"
      refers to** — a deterministic text→tower visual transform, not a machine-learning
      transformer model.
    - `codzeifyWord(s)` / `printCodze(z)` — gematria-style numeric encoding: letters A-Z (and
      Z-A reversed) are grouped into runs of 8, each letter's group-index becomes a digit, the
      digit string is read as a decimal integer and also rendered as binary. This is the
      **fingerprint** generator — a deterministic numeric signature of a word/string, computed
      both directions (AZ and ZA).
  - `VOIDONX` (repo root) — a saved worked example: real GPT-2 output text run through the
    squish/tower pipeline, showing the intended end-to-end shape (generate → squish → tower).
- `TOYBOK/COR.ipynb` — the original Jupyter notebook `pemdas.py`/`hollow.py` were extracted
  from. Check here first if a ported function's behavior is ambiguous — the notebook may have
  cells that didn't make it into the `.py` export.
- `TDW/`, `XPL/`, `docs/` — old GitHub Pages / static-site scaffolding, not relevant to the
  gpt2-alpine-c integration.
- `START_NOTEBOOK` — launches `jupyter notebook` with a raised iopub data rate limit (was
  needed for whatever large-output cells lived in `COR.ipynb`).

## Why it's here: the Apple-enrichment feature

Founder's request (2026-07-16): every Apple filed to IDUNA should carry, in addition to its
normal payload:
1. A GPT-2-generated fingerprint — text generated by the local model (via `gpt2-alpine-c`'s
   `scripts/serve.py` `/generate` endpoint, not this repo's TF1 code directly — that stack is
   dead/unsupported, this repo is reference only) run through this repo's squish/tower
   (`MTRXTWER`/`PRINTWR`) and gematria (`codzeifyWord`) transforms.
2. A model fingerprint — which model/checkpoint actually generated the content (hash or
   version string), so Apples remain honest about provenance the way `HQ-SPEC-AI-103`'s
   `fabledata` provenance fields require.
3. Astrological/transit info, always — using Dallas, TX as the reference location (the
   the server's own location (Dallas, TX), not the founder's; see memory `server-location-dallas`).

This is real backlog work, not yet built — see `EMILY/BACKLOG.md` for the tracking section
(added 2026-07-16, search "APPLE ENRICHMENT" or the section number noted in the same commit
that added this file). The squish/tower/gematria logic itself is simple enough to port directly
to Go rather than shell out to this old TF1 Python — treat this repo as the reference
implementation to port from, not a runtime dependency.
